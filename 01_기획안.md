## 1. 서비스 한눈에 보기

- **무엇:** 경기도볼링협회가 주최하는 대회의 **선수등록 → 팀편성 → 레인배정 → 점수입력 → 실시간 랭킹 노출**까지를 한 번에 처리하는 웹앱.
- **누가 보는가:**
  - **공개 사용자(로그인 없음):** 대회 결과 / 배정레인을 본다.
  - **관리자(로그인 필요):** 대회를 운영한다.
- **운영 흐름의 핵심:** 점수 입력 트래픽을 견디기 위해, **랭킹은 "게임 마감" 시점에만 1회 갱신**하고 그 결과를 캐시 테이블에 저장한다. 사용자 화면은 캐시만 본다.

## 3. 대회 상태(자동 산정)

대회 상태는 DB에 저장하지 않고 **`start_date`, `end_date`와 현재 날짜를 비교한 계산 값**으로 다룬다 (DB 뷰 또는 함수).

| 상태 | 조건 | 사용자 동작 |
|---|---|---|
| **예정(upcoming)** | `today < start_date` | 카드 클릭 시 "준비중입니다" 안내 모달, 진입 차단 |
| **진행중(ongoing)** | `start_date <= today <= end_date` | 결과 + 배정레인 모두 노출 |
| **종료(finished)** | `today > end_date` | 결과만 노출 (배정레인 탭 숨김) |

관리자가 강제 종료/연장이 필요한 경우를 위해 `status_override` 컬럼은 옵션으로 둔다(기본 NULL).

## 4. 사용자(공개) 페이지 흐름

```
[ 대회 목록 ]   ← 진입 시 기본 화면
      │
      ▼  (대회 카드 클릭)
[ 대회 상세 ]
   ├── 상태=예정: 모달 "준비중" → 뒤로
   ├── 상태=진행중: 종별 그리드 + (배정레인 탭) + (결과 탭)
   └── 상태=종료: 종별 그리드 + (결과 탭만)
      │
      ▼  (종별 카드 클릭)  예: 고등부 남자
[ 세부종목 선택 ]
      │   예: 개인전 / 2인조 / 3인조 / 5인조 중 등록된 것만
      ▼
[ 랭킹 화면 ]
   - 필터: 시/군 드롭다운
   - 검색: 소속/선수명 키워드
   - 표 컬럼: 순위, 시/군, 소속, 선수번호, 선수명, 1G~NG, 합계, 평균, 1위와의 핀차
   - 2/3/5인조는 개인행 + 팀행(밴드)이 같이 노출
   - 진행중 대회: 화면 상단에 "현재 X게임 진행중" 배지
```

### 4.1 대회 목록 화면

- 카드: 대회명 / 장소 / 기간 / 상태 배지 / 등록된 종별 칩들(예: `고등부남자` `중등부여자`)
- 정렬: 진행중 → 예정 → 종료(최근순)
- 모바일 우선 레이아웃, 1열 카드 리스트

### 4.2 대회 상세

- 종별이 카드 그리드로 노출 (예: `초등 U10 남자` `초등 U12 여자` `중등 남자` ...)
- 관리자가 등록한 종별만 노출, 종별별 등록 선수 수 같이 표시
- "배정레인 보기" 탭(진행중일 때만): 레인번호별로 선수/팀 카드를 그리드로 표시

### 4.3 세부종목 선택

- 종별 안에서 관리자가 등록한 세부종목만 노출 (개인전/2인조/3인조/5인조)
- 카드에 "총 N명 / N팀" 표시

### 4.4 랭킹 화면

#### 공통

- 컬럼: `순위`, `시/군`, `소속`, `선수번호`, `선수명`, `1G ~ NG`(N=대회 설정), `합계`, `평균`, `1위와의 핀차`
- 진행 중인 게임 컬럼은 회색 처리(아직 미마감)
- 동점자 동일 순위 처리: **최고게임 점수 높은 순** (구현은 ORDER BY total DESC, high_game DESC)
- 핀차 = `해당 행의 합계 - 1위의 합계` (음수, 1위는 0)

#### 개인전

- 단일 표. 페이지네이션 50명 단위.

#### 2인조 / 3인조

- 한 팀이 묶인 시각적 그룹(연한 배경 밴드)으로 표현
- 개인 점수 행들 위에 **팀 합계 행** 노출
  - 팀 행의 게임별 칸: 같은 게임에서 팀 멤버 점수의 **합**
  - 팀 행의 합계 = 팀 멤버 합계의 합
  - 팀 행의 평균 = (팀 합계) / (게임 수 × 인원 수)
- 팀 랭킹과 개인 랭킹은 토글(상단 탭)로 전환

#### 5인조

- 한 팀 안에 **최대 6명**의 로스터가 있고, 각 게임마다 **출전 5명**이 정해져 있음
- 개인 행: 6명 모두 노출, 게임별 점수 다 기록됨
- 팀 행: 각 게임마다 **그 게임의 출전 5명** 점수만 합산
- 시각적 표시:
  - 개인 행의 각 게임 칸에 작은 마크(▣ = 팀 합산 / ○ = 개인만)로 출전 여부 표시
  - 전반전(예: 1~2G) 출전 5명과 후반전(예: 3~4G) 출전 5명이 다를 수 있음

#### 필터 & 검색

- 시/군: 드롭다운(다중 선택 가능)
- 소속: 텍스트 부분일치
- 선수명: 텍스트 부분일치
- URL 쿼리에 상태 보존(공유 가능)

## 5. 관리자 페이지 흐름

### 5.1 인증

- `/admin/login` (이메일 + 비밀번호)
- 초기 시드 계정: `admin@bowling.com` / `admin123`, `role = super_admin`
- 로그인 후 `/admin` (대시보드)
- 추후 추가될 화면:
  - 비밀번호 변경 (`/admin/account`)
  - 관리자 계정 추가/삭제 (super_admin만, `/admin/users`)

### 5.2 사이드바 메뉴 (1차)

```
대시보드        — 진행중 대회 목록
대회 관리       — 대회 CRUD
선수 마스터     — 시/군·소속·선수 마스터
계정            — 내 정보, 비밀번호 변경
```

### 5.3 대회 생성/수정

| 입력 항목 | 비고 |
|---|---|
| 대회명 | 필수 |
| 장소 | 필수 |
| 기간 (시작/종료일) | 필수, 종료 < 시작 검증 |
| 사용 레인 범위 | 예: 3 ~ 10 (DB에는 정수 범위로) |
| 레인이동 방향 | 좌 / 우 |
| 레인이동 칸 수 | 정수(0이면 미사용) |
| 등록할 종별(다중) | 초등U10/초등U12/중등/고등/대학/일반 × 남/여 |
| 종별별 세부종목(다중) | 개인전/2인조/3인조/5인조 |
| 세부종목별 게임 수 | 기본 6. 5인조는 1~6 중 관리자 선택(드롭다운). |
| 5인조 전·후반 분기점 | 관리자가 직접 지정 — "전반전 끝 게임 번호". 예: 6게임이면 3(1~3G 전반, 4~6G 후반), 5게임이면 2(1~2G 전반, 3~5G 후반), 1게임이면 1(전반만, 교체 불가). |

> **노트:** 5인조 게임 수는 1~6 자유 선택. 분기점은 `1 ≤ split ≤ games_count` 범위에서 관리자가 직접 입력.
> `split == games_count`인 경우는 후반전이 없는 것으로 보고 교체 버튼 비활성화.

### 5.4 선수등록 (대회별)

#### 진입 흐름

```
[대회 상세 > 선수등록 탭]
  1) 시/군 선택 (고정 리스트)
  2) 소속 입력 (자유입력 + 자동완성: 같은 시/군의 기존 affiliations 추천)
  3) 선수명 입력란을 "추가" 버튼으로 늘려가며 한 번에 여러 명 입력
       - 마스터 선수와의 매칭: (시/군 + 소속 + 이름)이 동일하면 기존 player_id 재사용
       - 매칭 후보가 있으면 "이 선수가 맞나요?" 확인 후 선택
       - 없으면 새 player 생성
  4) "등록" 버튼 → 한 번에 여러 명 등록
```

#### 자동 처리

- **팀 라벨 자동 부여:** 같은 (시/군 + 소속)의 등록 순서대로 1~6번째는 `A팀`, 7~12번째는 `B팀`, 13~18은 `C팀` … 이후 수정/삭제 시 라벨 **재계산** (등록 순서 유지).
- **선수번호 자동 부여:** 대회 내에서 1부터 순차 부여. 삭제된 번호는 재사용하지 않음(공석으로 남김) → 출력물·기록 보존을 위해.
- **수정/삭제:** 가능. 단, 이미 점수가 입력된 선수는 "정말 삭제하시겠습니까? 점수가 함께 삭제됩니다" 경고.

#### 선수 검색

- 화면 상단 검색바: 시/군 필터 + 소속/이름 텍스트
- 표 컬럼: 선수번호, 시/군, 소속, 팀라벨, 이름, 매칭된 마스터 player_id

### 5.5 팀 편성 (2/3/5인조)

#### 화면 구조

- 종별 탭 → 세부종목 탭
- 좌측: 같은 (시/군+소속+팀라벨)별로 그룹화된 **미편성 선수 리스트** (체크박스)
- 우측: 이미 편성된 팀 카드 리스트 (멤버 / 해제 버튼)

#### 편성 방식 (두 가지 지원)

1. **클릭 방식**
   - 좌측에서 같은 (소속+팀라벨)인 선수 2명(2인조) / 3명(3인조) / 5명(5인조 출전선수)을 선택
   - "팀편성하기" 버튼 → 확인 모달

2. **번호 입력 방식**
   - 입력창에 `12,15`처럼 콤마로 선수번호 입력
   - "팀편성하기" → `"홍길동(12), 김철수(15)를 한 팀으로 편성하시겠습니까?"` 모달 → 확인

#### 검증 룰

- 같은 (시/군+소속+팀라벨) 안에서만 편성 가능. 다른 라벨이 섞이면 에러.
- 인원 수가 맞지 않으면 에러.
- 5인조의 경우 팀은 같은 (시/군+소속+팀라벨)의 **6명까지** 로스터로 묶고, 출전 5명은 게임별 출전 명단에서 정함(아래 5.7 참고).

#### 미편성 선수

- 편성에 들어가지 못한 선수(예: 같은 팀의 5번째 선수)는 그대로 둠 → 레인 배정에서 별도로 분산 배치, 점수는 개인 점수로만 기록.

### 5.6 레인 배정

#### 진입 흐름

```
[대회 상세 > 레인배정 탭] → (세부종목 선택)
  - 좌측: 사용레인 범위(대회 설정에서 가져옴, 변경 가능)
  - 중앙: 레인 카드 그리드 (3, 4, 5 … 10)
  - 우측: 미배정된 팀/선수 리스트
  - 상단: [랜덤 배정] [전체 초기화] 버튼
```

#### 랜덤 배정 알고리즘

> **원칙: 모든 레인에 거의 비슷한 인원이 가도록 균등 분산.** 6명은 하드 캡(절대 한도)일 뿐이며 무조건 채우지 않는다.

1. 사용레인 수 L 계산 (예: 3~10이면 8)
2. 팀(2/3/5인조) 단위로 셔플 → **한 팀은 반드시 같은 레인**
3. 미편성 선수도 셔플 → 남은 자리에 분산
4. **레인별 인원 수 균등화**: 모든 레인의 인원 수 차이가 **1을 넘지 않도록** 배정
   - 예: 사용레인 8, 총 인원 24 → 모든 레인 정확히 3명
   - 예: 사용레인 8, 총 인원 30 → 6개 레인 4명 + 2개 레인 3명 (편차 1)
   - 예: 사용레인 8, 총 인원 17 → 1개 레인 3명 + 7개 레인 2명 (편차 1)
5. **하드 캡: 한 레인 최대 6명**. 전체 인원이 `L × 6`을 초과하면 사용레인 부족 경고.
6. 팀 단위 배정으로 균등화가 정확히 안 떨어질 때(예: 5인조 5명 + 다른 1인) 차이는 최소화하되 같은 팀을 쪼개지 않는다.

#### 수동 배정

- 드래그앤드롭 (dnd-kit)
- 팀은 통째로 이동 (팀 카드를 드래그하면 멤버 같이 이동)
- 미편성 선수는 개별 카드 드래그
- 제약 위반(같은 팀이 분리되는 시도, 한 레인 초과)은 드롭 시 차단 + 토스트 안내

#### 레인이동규칙 미리보기

- 사용레인 + 이동 방향/칸 수 입력 후, 1G/2G/3G... 각 게임의 레인 매핑을 표로 미리보기
- 래핑 순환 적용 (예: 사용레인 3~10, 오른쪽 2칸, 1G 9번 → 2G 11 → 래핑하면 3번)
- **수식:** `lane_at_game(g) = ((base_lane - lane_start) + direction * offset * (g-1)) mod L + lane_start`
  - direction: +1(우) / -1(좌)

### 5.7 점수 입력

#### 진입 흐름

```
[대회 상세 > 점수입력 탭] → (세부종목) → (게임 번호)
  - 기본: 레인별 입력 (한 레인 = 한 페이지)
  - 토글: "전체 한번에 입력" (긴 페이지로 모든 레인 노출)
  - 각 게임의 헤더에 상태 배지: [입력중] / [마감됨]
  - 게임 마감 버튼: 그 게임을 잠그고 랭킹 캐시 갱신
  - 마감 후 수정은 super_admin만 가능 (이력 남김)
```

#### 입력 폼

- 행: 선수번호 + 이름 + 점수 입력 셀(integer, 0~300)
- 키보드 친화적: Tab/Enter로 다음 셀 이동
- 자동 저장: blur 또는 1초 디바운스 (개별 셀 단위로 UPSERT)
- 진행 중인 게임만 입력 가능 (마감된 게임은 잠금)

#### 게임 마감 → 랭킹 갱신

- 트랜잭션 단위:
  1. `game_states`에 `(event_id, game_number, locked=true, locked_at=now())` UPSERT
  2. `rankings` 캐시 테이블 전체 재계산 (해당 event_id)
  3. `team_rankings` 캐시 테이블 전체 재계산
- 사용자 화면은 `rankings` / `team_rankings` 만 조회 → 읽기 부담 최소화

#### 5인조 선수 교체 UI

- 게임 헤더 옆에 [선수교체] 버튼 (후반전 첫 게임 진입 직전에만 노출)
- 클릭 시 모달:
  - 6명 로스터 카드 (현재 출전 5명에 체크)
  - 교체할 선수 체크 변경 (예: E 빼고 F 넣기)
  - 확정 시:
    - 후반전 모든 게임의 `lineup`이 새 5명으로 설정
    - 교체out된 E의 레인은, 교체in된 F가 원래 있던 레인으로 자동 이동
    - F는 팀 레인으로 이동
  - 개인 점수는 모두 기록되지만, 팀 합산은 게임별 lineup 기준으로만 됨

## 6. 비즈니스 룰 모음 (구현 시 체크리스트)

### 팀 라벨

- 같은 (시/군 + 소속) 내 등록 순서로 6명마다 A, B, C ... 라벨 자동 부여
- 선수 삭제 시 라벨 재계산: 등록 순서를 유지하면서 빈 자리를 메우는 방식이 아니라, **남은 선수들의 등록 순서대로 다시 6명씩 묶기**
- 라벨 변경이 발생하면 이미 만들어진 팀 편성은 **검증 재실행** → 룰 위반이면 관리자에게 경고

### 선수번호

- 대회 단위 1부터 순차. 삭제된 번호는 **재사용 안 함**(공석으로 둠).
- 표시: zero-padding 없음(자연수). PDF/엑셀 출력 시 그대로 사용.

### 팀 편성 룰

| 세부종목 | 1팀 인원 |
|---|---|
| 개인전 | 1 (편성 없음) |
| 2인조 | 정확히 2명 |
| 3인조 | 정확히 3명 |
| 5인조 | 로스터 5~6명, 출전 5명 |

- 5인조에서 로스터가 4명 이하인 (시/군+소속+팀라벨) → 그 팀은 5인조 출전 불가 (에러 표시)
- 5인조 로스터가 **6명일 때만** 교체 가능 (5명만 있으면 출전은 가능하지만 교체 UI 비활성화)
- **교체 가능 조건:** 후반전이 존재해야 함 = `halftime_split_at < games_count`. 1게임이거나 split이 마지막 게임이면 교체 버튼 자체가 안 뜸.

### 미편성 선수의 레인 처리

- 같은 레인에 다른 소속의 미편성 선수가 같이 칠 수 있음 → **개인 점수만 기록**, 팀 합산 없음

### 동점자 처리

- ORDER BY `total DESC, high_game DESC, player_number ASC`
- 추후 룰 변경 가능성 고려해 `tournament.settings.tiebreaker_rule` 컬럼으로 보관 (기본 `high_game`)

### 핀차

- `1위와의 핀차 = 본인 합계 - 1위 합계` (음수)
- 1위는 0 표시

### 평균

- 개인: `합계 / 입력된 게임 수` (NULL인 게임은 제외)
- 팀: `팀 합계 / (입력된 게임 수 × 인원 수)` (5인조는 게임별 출전 인원 수 기준)

### 진행 상태 표시

- 게임이 `locked`되지 않으면 랭킹 표에서 해당 게임 칸은 회색 표시 (잠정 점수도 노출하지 않음)
- 사용자 화면은 **항상 마지막 마감된 게임까지의 결과**만 표시

## 7. 권한 / 보안

- 공개 페이지(사용자): 인증 없음. Supabase **공개 anon key**로 `rankings`, `team_rankings`, `lane_assignments` 등 **읽기 전용 뷰**만 접근.

- 관리자 변경 작업(점수 마감 등)은 `audit_logs` 테이블에 기록.

## 8. 성능/캐시 전략

| 케이스 | 전략 |
|---|---|
| 사용자 랭킹 조회 | `rankings`/`team_rankings` 캐시 테이블만 SELECT. 인덱스 `(event_id, rank)`. |
| 점수 입력 동시성 | 개별 셀 UPSERT. 마감은 트랜잭션 + advisory lock으로 순차화. |
| 게임 마감 시 재계산 | PostgreSQL 함수 `refresh_rankings(event_id)` 안에서 한 번에 처리. |
| 실시간 갱신 | Supabase Realtime을 `rankings` 테이블에 구독. 마감 시 한 번만 변경되므로 사용자 화면 fetch 횟수 최소. |
| 대회 목록 ISR | Next.js ISR(60s) 또는 SSR + revalidateTag로 캐싱. |

## 10. MVP 범위 vs 추후

### MVP (Phase 1)

- 대회 CRUD / 종별·세부종목 설정 / 레인 설정
